# PRISMA — Preferred Reporting Items for Systematic reviews and Meta-Analyses

*Encodes PRISMA 2020. A fast-start memory aid — confirm borderline calls against the live published checklist.*

## What it governs

**Systematic reviews and meta-analyses.** The transparency-and-reproducibility bar that separates a systematic review from a narrative opinion piece: a reader should be able to reproduce the search and see how every included study was found and judged.

## Highest-yield checklist items

1. **Search reproducibility** — the full search strategy: databases, exact terms/query, date ranges, and limits, for each source. If the search cannot be re-run from what's reported, the review is not systematic.
2. **PRISMA flow diagram** — records identified → screened → excluded (with reasons) → included, with numbers at every stage.
3. **Eligibility criteria pre-specified** — inclusion/exclusion defined a priori, ideally in a registered protocol (PROSPERO).
4. **Risk-of-bias assessment** — each included study appraised with a named tool (e.g., RoB 2, ROBINS-I, QUADAS-2), and how that assessment feeds the conclusions.
5. **Study selection process** — screened in duplicate / independently, with disagreement resolution.
6. **Synthesis method** — how results were combined; if meta-analyzed, the model and **heterogeneity** (I²) assessment; if not, why a narrative synthesis.
7. **Publication / reporting bias** — assessed (e.g., funnel plot) where appropriate.

## Review prompts

- Could you re-run this search from the paper alone? Are databases, terms, and dates all given?
- Is there a PRISMA flow diagram, and do the numbers reconcile from identification to inclusion?
- Was risk of bias formally assessed per study, and does it temper the conclusions — or is a high-bias study weighted equally?
- If results were pooled, was heterogeneity assessed, and is pooling appropriate given it?
